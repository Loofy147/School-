# Key Concepts for Product/UX System Design
# المفاهيم الأساسية لتصميم المنتج وتجربة المستخدم على مستوى النظام

This section covers the core methodologies for designing and defining a user-centric system.

- **[User Journeys](./user-journeys/index.md):** Mapping the user's path from start to finish.
- **[Service Blueprint](./service-blueprint/index.md):** Visualizing the entire service, front-to-back.
- **[Minimum Viable Product (MVP)](./minimum-viable-product/index.md):** Defining the smallest valuable product.
- **[Acceptance Criteria](./acceptance-criteria/index.md):** Formalizing the requirements for completion.
- **[End-to-End Scenarios](./end-to-end-scenarios/index.md):** Describing a complete user workflow.

---

يغطي هذا القسم المنهجيات الأساسية لتصميم وتعريف نظام يركز على المستخدم.

- **[رحلات المستخدم](./user-journeys/index.md):** تخطيط مسار المستخدم من البداية إلى النهاية.
- **[مخطط الخدمة](./service-blueprint/index.md):** تصور الخدمة بأكملها، من الأمام إلى الخلف.
- **[الحد الأدنى من المنتج القابل للتطبيق (MVP)](./minimum-viable-product/index.md):** تحديد أصغر منتج ذي قيمة.
- **[معايير القبول](./acceptance-criteria/index.md):** إضفاء الطابع الرسمي على متطلبات الإنجاز.
- **[سيناريوهات شاملة](./end-to-end-scenarios/index.md):** وصف سير عمل مستخدم كامل.
