# Key Concepts for Security Architecture & Threat Modeling
# المفاهيم الأساسية لهندسة الأمن ونمذجة التهديدات

This section covers the core principles and practices for designing a secure and resilient system.

- **[Threat Modeling](./threat-modeling/index.md):** Proactively identifying and mitigating potential threats.
- **[Defense in Depth](./defense-in-depth/index.md):** Using multiple, layered security controls.
- **[Principle of Least Privilege](./principle-of-least-privilege/index.md):** Granting only the minimum necessary permissions.
- **[Identity and Access Management (IAM)](./identity-and-access-management/index.md):** Managing who has access to what.
- **[Encryption](./encryption/index.md):** Protecting data both in-transit and at-rest.
- **[Secrets Management](./secrets-management/index.md):** Securely handling sensitive credentials.

---

يغطي هذا القسم المبادئ والممارسات الأساسية لتصميم نظام آمن ومرن.

- **[نمذجة التهديدات](./threat-modeling/index.md):** تحديد التهديدات المحتملة وتخفيفها بشكل استباقي.
- **[الدفاع في العمق](./defense-in-depth/index.md):** استخدام ضوابط أمنية متعددة ومتدرجة.
- **[مبدأ الامتياز الأقل](./principle-of-least-privilege/index.md):** منح الحد الأدنى من الأذونات اللازمة فقط.
- **[إدارة الهوية والوصول (IAM)](./identity-and-access-management/index.md):** إدارة من لديه حق الوصول إلى ماذا.
- **[التشفير](./encryption/index.md):** حماية البيانات أثناء النقل وفي حالة السكون.
- **[إدارة الأسرار](./secrets-management/index.md):** التعامل الآمن مع بيانات الاعتماد الحساسة.
