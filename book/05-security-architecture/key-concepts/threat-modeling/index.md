# Threat Modeling & STRIDE
# نمذجة التهديدات و STRIDE

Threat modeling is a proactive process where potential threats and vulnerabilities are identified, quantified, and mitigated during the design phase. A common framework for this is **STRIDE**, which provides a mnemonic for the different types of threats to consider.

- **[Spoofing](./spoofing/index.md):** Impersonating another user or service.
- **[Tampering](./tampering/index.md):** Modifying data without authorization.
- **[Repudiation](./repudiation/index.md):** Denying that an action was taken.
- **[Information Disclosure](./information-disclosure/index.md):** Exposing sensitive data.
- **[Denial of Service](./denial-of-service/index.md):** Making a system unavailable.
- **[Elevation of Privilege](./elevation-of-privilege/index.md):** Gaining unauthorized access.

---

نمذجة التهديدات هي عملية استباقية يتم فيها تحديد التهديدات ونقاط الضعف المحتملة وقياسها وتخفيفها أثناء مرحلة التصميم. الإطار الشائع لهذا هو **STRIDE**، والذي يوفر تذكيرًا للأنواع المختلفة من التهديدات التي يجب مراعاتها.

- **[الانتحال](./spoofing/index.md):** انتحال شخصية مستخدم أو خدمة أخرى.
- **[العبث](./tampering/index.md):** تعديل البيانات دون إذن.
- **[الإنكار](./repudiation/index.md):** إنكار اتخاذ إجراء ما.
- **[الكشف عن المعلومات](./information-disclosure/index.md):** كشف البيانات الحساسة.
- **[رفض الخدمة](./denial-of-service/index.md):** جعل النظام غير متاح.
- **[تصعيد الامتيازات](./elevation-of-privilege/index.md):** الحصول على وصول غير مصرح به.
