# End-to-End (E2E) Tests
# اختبارات من طرف إلى طرف (E2E)

These tests simulate a full user journey through the application. They are slow, brittle, and expensive, so you should have very few of them. For example, testing that a user can log in, add an item to their cart, and check out.

---

تحاكي هذه الاختبارات رحلة مستخدم كاملة عبر التطبيق. إنها بطيئة وهشة ومكلفة، لذلك يجب أن يكون لديك عدد قليل جدًا منها. على سبيل المثال، اختبار قدرة المستخدم على تسجيل الدخول وإضافة عنصر إلى عربة التسوق الخاصة به وتسجيل الخروج.
